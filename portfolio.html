Since 2018, I have been working at IBM in the z/TPF Development Lab. I develop and maintain the following database access middleware for the z/TPF operating system:
- z/TPF support for MongoDB: An interface to legacy z/TPFDF databases which allows remote clients to access legacy data on z/TPF using MongoDB client libraries
- z/TPF file system support: Support for a POSIX-compliant file system with additional functionality for the z/TPF system, including process and system pseudo-file systems
- z/TPF logical record cache support: Support for in-memory caches which can be accessed by any application running on a z/TPF system

Prior to working with IBM, I worked as an independent contractor for HireWare. At HireWare, I contributed designs and prototypes in a fast-paced startup environment in the staffing industry. As the sole developer on the team, I learned how to navigate an existing Ruby on Rails code base and refined the existing software into a prototype of a minimum viable product.

I was granted a Bachelor of Science in Computer & Information Sciences at the State University of New York at Potsdam in 2017. In my undergraduate career, I became familiar with computing concepts including algorithm analysis, operating system design, and agile methodologies. I served as a teaching assistant in the Computer & Information Sciences department, assisting students with laboratory assignments and reinforcing concepts taught in class.

Technical projects

z/TPFDF remote subfile support

z/TPFDF remote subfile support provides the capability to store data for z/TPFDF subfiles in remote data stores while z/TPFDF applications continue to read and update remote subfiles as if the data was stored locally on the z/TPF system.

As a developer for this project, I designed and implemented code that transforms binary data in an arbitrary format into BSON (Binary JSON), and transmits the BSON data to a MongoDB server running on a remote system.

I also contributed to the design and implementation of graceful fault tolerance for this system.

In 2020, IBM began a concerted effort to replace racially and culturally discriminatory language with more inclusive language across code and content.

z/TPF user security database individual role management

z/TPF user security database individual role management provides the capability to add or remove individual roles for a user in the z/TPF user security database.

I contributed significantly to the design of this enhancement and served as technical mentor to the developer that implemented the enhancement.
